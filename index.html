Understanding the "Launch Command"

The Launch Command is the most crucial piece of information for each program. It's the exact instruction your computer needs to find and run your chosen program or script. Think of it like a very specific address for your application.

If this command is incorrect, the Hub won't be able to open your program (though in this web demo, it only simulates launching).

When you use the "Select File to Pre-fill" option, it will put the filename in the Launch Command field. This is often NOT enough. You usually need the full path to the file, or a command that's recognized by your system's command line (Terminal or Command Prompt).

How to Find the Launch Command:

For Windows (.exe files, .bat scripts):
  • Find the File: Locate the program's main executable file (usually ends in .exe) or your script file (e.g., .bat, .py) in File Explorer.
  • Copy Path:
    • Hold down the Shift key, then right-click on the file.
    • Select "Copy as path".
  • Paste: Paste this copied path into the "Launch Command" field. It will look something like: "C:\Program Files\MyAwesomeApp\MyApp.exe" (the quotes are important if the path has spaces).
  • For Python scripts: You might need python "C:\MyScripts\myscript.py" or if Python is in your PATH, just python myscript.py (if the script is also in a PATH directory or you run the Hub from the script's directory).
For macOS (.app applications, scripts):
  • Applications (.app):
    • The simplest command is usually open -a "Application Name". For example, to open TextEdit, the command is open -a "TextEdit".
    • Alternatively, find the application in Finder (usually in the /Applications folder). The path would be like: /Applications/Visual Studio Code.app. To run it, you might use open "/Applications/Visual Studio Code.app".
  • Scripts (e.g., .sh, .py):
    • Find your script file in Finder.
    • Open Terminal (you can find it in Utilities or search with Spotlight).
    • Drag the script file from Finder directly into the Terminal window. This will paste its full path. Copy this path.
    • For Python scripts, it might be python3 /Users/yourname/Documents/myscript.py. For shell scripts, it might be /Users/yourname/Documents/myscript.sh (ensure it's executable: chmod +x /path/to/script.sh).
For Linux (executables, scripts):
  • Find the File: Locate your executable file or script.
  • Get Path: Open your terminal, navigate to the directory containing the file, and type pwd (print working directory), then add /filename. Or, right-click the file in your file manager and look for an option like "Copy Path" or "Properties" to find its location.
  • Paste: The path might look like /home/yourusername/MyPrograms/my_app or /usr/bin/some_program.
  • Scripts: For Python, it's often python3 /home/yourusername/scripts/myscript.py. For shell scripts, /home/yourusername/scripts/myscript.sh (ensure it's executable: chmod +x /path/to/script.sh).
  • If in PATH: If the program is installed system-wide and its command is in your system's PATH environment variable (e.g., firefox, gimp), you can often just use the program's name as the command.
